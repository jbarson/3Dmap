## 2024-03-25 - Content Security Policy (CSP) Gap in index.html

**Vulnerability:** The `index.html` file was missing a `script-src` directive in its `Content-Security-Policy` meta tag. In addition, Vite dev server features like HMR (`ws:`) and inline styles (`unsafe-inline`) were not accounted for, nor was `blob:` allowed for Three.js textures in `img-src` in the main view.
**Learning:** Without `script-src`, CSP falls back to `default-src`. While `default-src 'self'` provides baseline protection, it is a security best practice to explicitly declare `script-src 'self'` to prevent accidental inline scripts or eval if the default changes or if another developer misconfigures `default-src`. In addition, proper CSP rules for modern frameworks (Vite) and 3D libraries (Three.js) must be carefully crafted to avoid breaking local development while remaining secure in production.
**Prevention:** Always explicitly define `script-src 'self'` and configure `style-src 'unsafe-inline'`, `connect-src ws:`, and `img-src blob:` securely for Vite and Three.js environments.
